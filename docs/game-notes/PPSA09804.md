## Benchmark

The benchmark measures cold boot-to-terminal-state latency, not frame rate.
Each Release run used a 15-second timeout, import tracing, and the same local
input. All three runs ended in the same access violation at the unresolved
Unity/IL2CPP function pointer.

| Metric | Run 1 | Run 2 | Run 3 | Aggregate |
|---|---:|---:|---:|---:|
| Duration | 2.446 s | 2.422 s | 2.167 s | 2.345 s average |
| Peak working set | 170.9 MiB | 183.5 MiB | 162.0 MiB | 183.5 MiB maximum |
| Peak private memory | 172.8 MiB | 192.7 MiB | 162.1 MiB | 192.7 MiB maximum |
| Peak CPU time | 1.062 s | 1.125 s | 0.969 s | 1.052 s average |
| Highest import | 2,736 | 2,736 | 2,736 | Deterministic |
| First frame | No | No | No | No |

Before the branch changes, the title failed at the raw entry address after
discovering zero imports. It now consistently discovers 583 NIDs, processes
29,556 relocations, and reaches import 2,736. The import milestone is the
meaningful comparison; the short duration ends at a known crash and is not a
game-performance result.

## Observed Behavior

The executable loads and begins Unity initialization without opening a window.
It reads its boot data, configures libc state, reserves the expected virtual
memory pools, and queries mouse and user-service capabilities. Unity requests
hundreds of IL2CPP API addresses through `il2cpp_api_lookup_symbol`. Since
those addresses are unavailable, a stored error value is eventually executed
as code. The run is deterministic across all three benchmark samples.
